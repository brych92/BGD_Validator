SF1 — Перевірка можливості відкриття файлу

Що означає:
Валідатор намагається відкрити кожний переданий файл за допомогою бібліотеки GDAL/OGR. Якщо бібліотека повертає None, це означає, що файл неможливо прочитати — імовірно він пошкоджений, має неправильний формат або недоступний за вказаним шляхом.

Чому це важливо:
Якщо файл не відкривається, жодні подальші перевірки структури, геометрії чи атрибутів неможливі. Такий файл блокує робочий процес і може приховувати критичні помилки даних.

Як виправити:
• Переконайтесь, що шлях до файлу правильний і уникає нестандартних символів або надто довгих директорій.
• Спробуйте відкрити файл в QGIS або іншій GIS‑програмі — якщо він не відкривається, експортуйте шар у новий файл того самого формату.
• Перевірте, чи відповідає розширення файлу його реальному формату (наприклад, .gpkg, .shp, .geojson).
• Скопіюйте файл на локальний диск, щоб виключити проблеми з мережевими чи хмарними сховищами.
• За потреби скористайтесь інструментом ogrinfo для діагностики або повторно експортуйте дані з бекапу.

Алгоритм перевірки:
1. Отримати абсолютний шлях до файлу.
2. Викликати ogr.Open(path, 0) у режимі «тільки читання».
3. Якщо функція повертає об’єкт DataSource — файл коректний, перевірка успішна.
4. Якщо повертається None — зафіксувати критичну помилку SF1 і позначити файл як пошкоджений.
