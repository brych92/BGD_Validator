SLF3 Перевірка наявності всіх атрибутів, заданих структурою

Що означає:
Шар не містить одного або декількох атрибутів, які передбачені структурою даних. Наприклад, структура вимагає поле kod, а в шарі його немає.

Чому це важливо:
Автоматизовані процеси — такі як стилізація, звіти, фільтри, зовнішні API або модулі публікації — розраховують на наявність строго визначених полів. Їхня відсутність унеможливлює подальшу роботу або призводить до помилок.

Як виправити:
Уточніть список обов’язкових полів у структурі. Додайте відсутні атрибути вручну або через інструмент Refactor Fields у QGIS. Для полів з обов’язковим типом — одразу встановіть правильний тип даних (текст, число, дата тощо).

Алгоритм перевірки:
1. Отримується список атрибутів, що вимагаються структурою.
