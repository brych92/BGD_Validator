SL5 — Перевірка унікальності значення атрибута

Що означає:
У зазначеному атрибутивному полі шару (наприклад, OBJECTID, kadnum, global_id) виявлено повторювані значення. Відповідно до структури даних, це поле має містити унікальне значення для кожного об’єкта, тому дублікати вважаються помилкою.

Чому це важливо:
Неунікальні ідентифікатори або інші атрибути, для яких вимагається унікальність, руйнують логіку пов’язаних процесів, зокрема:

  • помилки під час приєднання таблиць (JOIN) або створення представлень, що спираються на первинний ключ;
  • баги в WEB‑сервісах і API — запити повертають множину об’єктів замість одного, або кидають duplicate key;
  • збій аналітики (ретейси, підрахунок площ/довжин), оскільки розрахунки ведуться за некоректним ключем;
  • неможливість публікувати шар у державні реєстри, де перевірка унікальності — обов’язкова.

Як виправити:

  1. У QGIS відкрийте Таблицю атрибутів "Умовне форматування" (Conditional Formatting). Оберіть атрибут, в якому потрібно перевірити унікальність (наприклад, kadnum, OBJECTID). Натисніть "+", щоб додати нове правило форматування. У полі вираз введіть: count("field", group_by:="field") > 1 (де field – замініть на потрібну назву атрибуту). Оберіть стиль (наприклад, червоне тло або жирний шрифт) — щоб виділити ним дублікати. Натисніть OK.
  2. Вирішіть, який об’єкт слід залишити, а для яких задати інші унікальні значення атрибуту.
  3. Для автоматичного виправлення можна скористатися Processing ➜ Delete duplicate geometries або Delete duplicates by attribute або Python скриптом, який генерує новий унікальний ID чи інше значення атрибуту.
  4. Після коригування повторно запустіть перевірку SL5, щоб переконатися у відсутності дублікатів.

Алгоритм перевірки:

  1. Зі структури отримується перелік полів із вимогою «unique=true».
  2. Для кожного такого поля будується множина значень, під час ітерації підраховується кількість повторів.
  3. Якщо знайдено дублікати — у результати записується список неунікальних значень і їхня кількість; помилці присвоюється критичність 2 (Висока).
  4. Якщо дублікатів немає — перевірка повертає «OK», критичність 0 (Проблем не виявлено).
