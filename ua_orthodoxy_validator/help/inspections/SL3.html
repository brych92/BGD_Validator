SL3 — Перевірка системи координат шару

Що означає:
Система координат (CRS) шару не входить до переліку дозволених для обраної структури. Наприклад, очікувані EPSG:5564 або EPSG:9864, а а фактично шар має інший код системи координат.

Чому це важливо:
Невірна CRS спричиняє геометричний зсув — шари «лягають» у різних/хибних місцях карти, аналітичні запити повертають нульовий чи помилковий перетин, а при експорті до державних реєстрів такі дані можуть бути автоматично відхилені.

Як виправити:
Визначте правильну систему координат для шару:
• якщо геометрія вже у потрібній проєкції, просто перевизначте CRS шару через Властивості(Layer) → Джерело(Source) → Задати систему координат(Set coordinate reference system).
• якщо геометрія у іншій проєкції, перепроєктуйте шар: Меню Вектор (Vector) → Геообробка(Geoprocessing) → Перепроєктувати шар(Reproject layer), обравши, наприклад, EPSG:5564 або EPSG:9864.
Збережіть результат і переконайтесь, що шар відкривається без попереджень та правильно суміщується з іншими шарами або/та базовою картою.

Алгоритм перевірки:
1. Зчитується фактична CRS шару.
2. Із метаданих/структури отримується перелік дозволених кодів (наприклад, 5564, 9864).
3. Виконується порівняння: чи входить CRS шару до списку.
4. Якщо так — помилки немає; якщо ні — формується SF3‑повідомлення з указанням кодів: [фактичний, дозволені].
5. Критичність встановлюється висока, оскільки відхилення робить подальший аналіз некоректним.
