SL3 — Перевірка системи координат шару

Що означає:
Система координат (CRS) шару не входить до переліку дозволених для обраної структури. Наприклад, очікувані EPSG:5564 або EPSG:9864, а фактично шар має інший код.

Чому це важливо:
Невірна CRS спричиняє геометричний зсув — шари «лягають» у різних місцях карти, аналітичні запити повертають нульовий чи помилковий перетин, а при експорті до державних реєстрів такі дані можуть бути автоматично відхилені.

Як виправити:
Визначте правильну систему координат для шару:
• якщо геометрія вже у потрібній проєкції, просто перевизначте CRS шарові через Властивості → Джерело → Задати систему координат.
• якщо геометрія у іншій проєкції, перепроєктуйте шар: Меню Вектор → Геообробка → Перепроєктувати шар, обравши, наприклад, EPSG:5564 або EPSG:9864.
Збережіть результат та переконайтеся, що він відкривається без попереджень і правильно суміщується з іншими шарами.

Алгоритм перевірки:
1. Зчитується фактична CRS шару.
2. Із метаданих/структури отримується перелік дозволених кодів (наприклад, 5564, 9864).
3. Виконується порівняння: чи входить CRS шару до списку.
4. Якщо так — помилки немає; якщо ні — формується SF3‑повідомлення з указанням кодів: [фактичний, дозволені].
5. Критичність встановлюється висока, оскільки відхилення робить подальший аналіз некоректним.
