SLF4 Перевірка наявності всіх обов’язкових атрибутів

Що означає:
У структурі даних зазначено, що певні атрибути є обов’язковими для заповнення. Проте відповідні атрибути (поля) у шарі відсутні. Це не дає змоги користувачу навіть вручну внести потрібну інформацію або скористатися формою введення.

Чому це важливо:
Відсутність обов’язкових атрибутів (полів) унеможливлює повноцінне заповнення даних. Це може спричинити помилки під час публікації, відправлення до реєстру або подальшої аналітичної обробки. Така перевірка є базовим етапом контролю структури даних.

Як виправити:
Уточніть перелік обов’язкових атрибутів (полів) згідно зі структурою даних. Додайте відсутні атрибути через таблицю атрибутів або властивості шару (Layer Properties → Fields Properties). Обов’язково одразу встановіть для кожного атрибута коректний тип даних (текст, число, дата тощо).

Алгоритм перевірки:
1. Зі структури обираються лише ті атрибути, де вказано вимогу обов’язкового заповнення.
2. Порівнюється з поточним списком полів шару.
3. Якщо якесь з обов’язкових полів відсутнє — створюється повідомлення з критичністю 2 (Висока).
4. Якщо всі поля присутні — критичність 0 (помилок не виявлено).
