Перевірка типу даних атрибута

Що означає:
Тип даних поля (атрибута) у шарі не відповідає типу, зазначеному в структурі. Наприклад, поле має тип Text, але згідно вимог має бути Integer.

Чому це важливо:
Типи даних визначають, як з атрибутами можна працювати. Якщо тип вказано неправильно:

  • неможливо виконувати математичні операції або сортування;
  • збільшується ризик помилок при фільтрації, генерації звітів чи формуванні шарів-похідних;
  • при експорті в інші формати (наприклад, у базу даних або JSON) дані можуть бути некоректно оброблені.

Як виправити:
У QGIS скористайтеся Field Calculator або Refactor Fields для створення нового поля з правильним типом, або ж перезапишіть існуюче. Після цього — видаліть старе поле або збережіть результат в окремий шар.

Алгоритм перевірки:
1. Зчитується тип поля з шару.
2. Зіставляється з очікуваним типом, який задано у структурі.
3. Якщо тип не збігається — фіксується помилка.
4. Присвоюється висока критичність (2), оскільки відхилення у типі може суттєво вплинути на подальше використання даних.
